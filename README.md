# [智慧型待辦事項優先排程器]

## Proposal Report

### 動機與目標
因為我自己是一個很沒有執行力的人，常常事情很多但我會沒有頭緒要先做哪一個，或是分不清楚哪件事情比較重要需要事先處理的，所以想要做出一個能夠幫我排定優先順序的工具並以此來督促我去完成，每次只顯示一項任務會讓我有最明確的目標可以去執行，希望可以藉此提高我的執行力。

### 預期功能
任務新增與屬性設定：使用者可輸入任務名稱、預估耗時、緊急程度與重要程度。
動態權重計算：系統根據任務屬性與自訂公式，自動計算出每個任務的「優先權分數」。
自動排程與置頂推薦：過濾掉繁雜的清單，系統介面永遠將當下「最高優先權」的任務置頂，引導使用者一次專注完成一件事。
時間逼近自動升級：隨著截止日期逼近，原本低優先級未完成的任務，其分數會隨時間函數自動提升。

### 使用技術
程式語言：C++ 
架構設計：物件導向設計 (OOP)
資料儲存：文字檔或 JSON 檔案輸入輸出 (File I/O)，達成任務資料的持久化。
使用者介面：命令列互動介面 (CLI)。

### 時程規劃
第一週：系統架構設計，定義 `Task` 類別屬性與權重計算的數學公式。
第二週：實作核心資料結構 (Priority Queue / Max-Heap) 以及基本的任務新增、刪除功能。
第三週：實作動態分數更新機制，以及檔案讀取與儲存 (File I/O) 功能。
第四週：邊界測試 (Edge Cases Test)、除錯 (Debugging)，並整理程式碼。
第五週：撰寫專案 README 文件，並發布 GitHub Release。

### 與課程的關聯
資料結構的應用：此次專題的核心是 優先權佇列 與 最大堆積。若使用一般陣列尋找最高優先級任務需耗費 O(n) 的時間，但透過自行實作 Max-Heap，能將新增任務與取出最高優先級任務的時間複雜度完美降至 O(log n)，實際應用了課程中探討的樹狀結構與效能優化概念。
進階程式設計：深度運用 物件導向 (OOP) 的封裝特性來管理任務，並會實作 運算子多載，讓程式知道如何依據自訂的演算法公式來比較兩個任務物件的優先大小。同時也會涵蓋指標操作與檔案處理的技術。
